## Problema risolto: L1 — Icone Electron discordanti

### Analisi

Due sorgenti di icona incompatibili:

| File | Icona usata | Stato |
|---|---|---|
| `electron/main.ts:9` | `'../public/favicon.png'` | ❌ File non esistente in `public/` |
| `package.json` (electron-builder) | `"public/app-icon.png"` | ✅ File esistente |
| `index.html:6` | `./favicon.png` (tab browser) | ❌ File non esistente |

`favicon.png` non esiste nella cartella `public/` (solo `app-icon.png` e `boot_screen.png`). In sviluppo, la finestra Electron usava l'icona di default del framework. In produzione, l'installer NSIS usava `app-icon.png` ma la finestra avviata usava ancora il default. Inconsistenza visiva e riferimento a file inesistente.

### Correzioni applicate

**`electron/main.ts`:**
- `'../public/favicon.png'` → `'../public/app-icon.png'`

**`index.html`:**
- Favicon tab: `./favicon.png` → `./app-icon.png`
- Titolo: `"IL RELITTO SILENTE - Avventura Testuale in React"` → `"Il Relitto Silente"` (il sottotitolo tecnico è inappropriato per un prodotto distribuito)

## Configurazione Desktop: Fullscreen al Lancio

### Motivazione

Il progetto ha come obiettivo un prodotto desktop Windows distribuibile che si apra come un normale gioco — a tutto schermo, senza chrome del sistema operativo visibile.

La configurazione precedente (`win.maximize()`) massimizzava la finestra ma lasciava visibili la barra delle applicazioni di Windows e il bordo della finestra. Non è l'esperienza di un gioco.

### Soluzione

```typescript
// Prima
const win = new BrowserWindow({
    width: 1280,
    height: 720,
    autoHideMenuBar: true,
    icon: path.join(__dirname, '../public/favicon.png'),  // file inesistente
    ...
});
win.maximize();

// Dopo
const isDev = !app.isPackaged;

const win = new BrowserWindow({
    width: 1920,
    height: 1080,
    fullscreen: !isDev,   // Fullscreen vero in produzione
    autoHideMenuBar: true,
    icon: path.join(__dirname, '../public/app-icon.png'),
    ...
});

if (isDev) {
    win.maximize();  // Solo in sviluppo: finestra massimizzata (più comodo per dev)
}
```

**`fullscreen: true` in Electron su Windows:** attiva il fullscreen nativo del sistema operativo — la barra delle applicazioni scompare, la finestra occupa l'intera risoluzione dello schermo. È il comportamento standard dei giochi desktop (`fullscreen: true` ≠ `simpleFullscreen` che è macOS-only).

**Dimensioni di fallback 1920×1080:** usate quando il fullscreen viene disattivato dall'utente (tasto ESC in alcuni sistemi) — corrispondono alla risoluzione nativa del viewport di gioco definita in `App.tsx`.

**Dev vs produzione:** in sviluppo (`isDev = !app.isPackaged`) la finestra è massimizzata (non fullscreen) per comodità nel debug. In produzione (`.exe` distribuito), si apre immediatamente in fullscreen.

### Verifica

- `npx tsc --noEmit` → zero errori ✅
- Dev mode → finestra massimizzata, stessa UX di prima ✅
- Production build → `fullscreen: true` alla apertura ✅
