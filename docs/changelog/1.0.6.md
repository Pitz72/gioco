## Problema risolto: GR7 — Placeholder non sostituiti nel prodotto finale

### Analisi

Due file distribuiti agli utenti finali contenevano testi segnaposto mai sostituiti con valori reali.

**`LEGGIMI.txt` riga 21:**
```
(c) 2025 - Tuo Nome / Studio
```
Questo testo era visibile direttamente dall'utente all'apertura del file incluso nell'installer, comunicando una mancanza di cura nel prodotto finale. In aggiunta, l'anno copyright era `2025` invece di `2026`, e la versione indicata nell'header era `v1.0.1` (ormai superata).

**`package.json` campo `build.appId`:**
```json
"appId": "com.tuonome.relittosilente"
```
L'`appId` viene usato da `electron-builder` per identificare l'applicazione a livello di sistema operativo (registro di Windows, bundle ID su macOS, desktop entry su Linux). Un `appId` con `tuonome` nel nome è visibile nei log di installazione, nel registro di sistema e nei metadati dell'app installata.

### Correzioni applicate

**`LEGGIMI.txt`:**
- Riga 2: `v1.0.1 - Gold Master` → `v1.0.6`
- Riga 21: `(c) 2025 - Tuo Nome / Studio` → `(c) 2026 - Il Relitto Silente Project`

**`package.json`:**
- `"appId": "com.tuonome.relittosilente"` → `"appId": "com.relittosilente.adventure"`

L'`appId` `com.relittosilente.adventure` è conforme al formato DNS-reverse standard per identificatori applicazione, non contiene placeholder e identifica chiaramente il progetto.
