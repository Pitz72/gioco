## Modifiche

### 1. Boot screen — colore fosforo esatto — `BootScreen.tsx`

**Problema:** Il filtro `sepia(1) hue-rotate(90deg) saturate(3.5)` produceva un verde leggermente diverso da `--p-main` (#33ff00): il fosforo del boot appariva di tonalità differente rispetto al testo di gioco.

**Soluzione:** Tecnica `grayscale + mix-blend-mode: multiply`:
- L'immagine viene convertita in scala di grigi con `filter: grayscale(1) contrast(1.5) brightness(X%)`
- Un `div` sovrapposto con `background: #33ff00` e `mix-blend-mode: multiply` tinge i pixel in modo matematicamente esatto: bianco × #33ff00 = #33ff00, nero × qualsiasi = nero
- Il risultato è il fosforo P1 identico a `--p-main` su tutta la bitmap
- Il `clipPath: inset(0 0 ${100 - progress}% 0)` viene applicato a entrambi i layer in sincronia

### 2. Sistema salvataggio in-game a slot — `SaveLoadOverlay.tsx` (nuovo) + `App.tsx`

Sostituisce completamente le finestre OS (dialog di sistema Windows) con un overlay stile terminale nativo al gioco.

**`SaveLoadOverlay.tsx` — nuovo componente:**
- 5 slot persistenti in `localStorage`, chiave `relitto-slot-{id}`
- `SlotData: { playerState, savedAt: string, location: string }` — anteprima con nome stanza e data/ora
- Overlay modale su `position: absolute; inset: 0` nell'area schermo, non rompe il layout
- Gestione tastiera con `{ capture: true }` + `e.stopImmediatePropagation()` — intercetta prima di qualsiasi altro handler (incluso il gestore F-key globale)
- Tasto `ESC` per annullare; tasti `1–5` per selezionare lo slot
- Mouse hover con feedback visivo; messaggio di errore per slot vuoto in modalità load (auto-dismiss 2500ms)
- Stile: finestra DOS con barra titolo fosforo pieno, lista slot con badge numerati, footer con istruzioni

**`App.tsx` — integrazione:**
- Rimossi `saveGame` / `loadGame` (vecchi callback che aprivano dialog OS)
- Aggiunto stato `slotMenu: 'save' | 'load' | null`
- Aggiunti callback: `openSaveMenu`, `openLoadMenu`, `handleSlotSave`, `handleSlotLoad`, `handleSlotCancel`
- `handleFKey`: F3 chiama `openLoadMenu()`, F4 chiama `openSaveMenu()`
- `submitCommand`: comandi `salva`/`carica` aprono i menu slot invece dei vecchi dialog
- `FunctionKeyBar`: F3/F4 `onClick` aggiornati
- `<SaveLoadOverlay>` renderizzato nell'area schermo quando `slotMenu !== null`, con `position: absolute` che lo sovrappone al contenuto senza spostare il layout

### 3. Cursore parser a posizione 0 — `CommandLine.tsx`

**Problema (v1.0.18):** Il cursore lampeggiante appariva dopo il testo placeholder "Cosa fai?" invece di essere sovrapposto alla prima lettera, come nei veri terminali anni '80.

**Soluzione:** Quando il campo è vuoto, il cursore usa `position: absolute; left: 0; top: 50%; transform: translateY(-50%); zIndex: 2` dentro un contenitore `position: relative`. Il testo "Cosa fai?" rimane sotto con `zIndex: 1`. Quando il cursore è acceso copre la "C", quando si spegne la lettera è visibile — comportamento identico ai terminali VT100/C64.
