## Problema risolto: L3 — Google Fonts da CDN, non funziona offline

### Analisi

`index.html` caricava il font `Press Start 2P` da Google Fonts CDN:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Press+Start+2P&display=swap" rel="stylesheet">
```

Per un'applicazione web questo è accettabile. Per un gioco desktop distribuito come `.exe`, è un problema:
- **Senza connessione internet**, il font non carica: il fallback è `monospace` generico, che rompe completamente l'estetica retro del gioco
- Electron carica i file locali ma non può garantire l'accesso a Google's CDN
- Viola l'obiettivo di "applicazione desktop offline-first"
- Inviare richieste a `fonts.googleapis.com` ad ogni avvio dell'app è un comportamento inatteso per un gioco installato localmente

### Correzione

**1. Installato `@fontsource/press-start-2p`:**
```bash
npm install @fontsource/press-start-2p
```

`fontsource` è il pacchetto ufficiale della community per distribuire Google Fonts come moduli npm self-hosted. Include i file `.woff2` direttamente in `node_modules/`. Vite li bundla nel build di produzione.

**2. Import aggiunto in `index.css`:**
```css
@import "tailwindcss";
@import "@fontsource/press-start-2p";
```

Il font viene importato nel foglio di stile principale. Vite processa l'import, copia i file `.woff2` in `dist/assets/` e aggiorna il CSS con i path corretti e hash per cache busting.

**3. Rimossi i link Google Fonts CDN da `index.html`:**
```html
<!-- Rimossi: -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Press+Start+2P&display=swap" rel="stylesheet">
```

L'applicazione non effettua più nessuna richiesta di rete al lancio. Il font `'Press Start 2P'` riferito in `index.css` (`font-family: 'Press Start 2P', monospace`) continua a funzionare identicamente — la dichiarazione del nome font nel pacchetto `@fontsource` è identica a quella di Google Fonts.

### Impatto sul bundle

Il file WOFF2 di `Press Start 2P` è ~32KB. Nel build di produzione viene incluso in `dist/assets/` con hash, caricato una sola volta al primo render e poi dalla cache del browser/Electron. Nessun impatto percettibile sulle performance.

### Verifica

- `npx tsc --noEmit` → zero errori ✅
- Font caricato localmente da `node_modules/` → funziona offline ✅
- Nessuna richiesta a `fonts.googleapis.com` ✅
- Aspetto visivo invariato ✅
