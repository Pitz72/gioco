## M1 — Precisazione comportamento localStorage in STATO_PROGETTO.md

### Problema
`docs/sviluppo/STATO_PROGETTO.md` sezione 8 (debito tecnico residuo) dichiarava che lo slider volume SFX «aggiorna localStorage». Questa descrizione era fuorviante: `storageService.ts` usa localStorage **solo come fallback** per la modalità web pura (`npm run dev` senza Electron). In produzione Electron, tutte le preferenze audio transitano via IPC e vengono scritte in `<userData>/settings.json`.

### File modificato
- `docs/sviluppo/STATO_PROGETTO.md` — nota sullo slider SFX riscritta per distinguere il comportamento in produzione (IPC filesystem) dal fallback web (localStorage).
