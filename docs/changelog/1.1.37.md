# v1.1.37 — Bugfix: attivazione Cristallo Dati promossa a comando globale

## Problema
`activateCrystalCommand` era un Room Command di `corridoioPrincipale.ts`.
"USA DISPOSITIVO SU CRISTALLO" funzionava solo nel Corridoio Principale.
Il giocatore che raccoglieva entrambi gli item nell'Arca Biologica e tentava
di attivarli lì riceveva un fallimento silenzioso (il targetItem Cristallo
era in inventario, non nella stanza, e l'item system non lo trovava).

## Fix

### gameLogic.ts
Aggiunto handler globale in `processCommand`, dopo i comandi globali
(inventario, mappa, ecc.) e prima dei Room Commands. Usa regex identico
all'originale + variante con `attiva`:

```
/^(usa|attiva) (dispositivo|dispositivo medico|strumento) (su|con) (cristallo|cristallo dati|cristallo dati opaco)$/
```

Controlla solo l'inventario (non la stanza). Gestisce [PAUSE] manualmente
(indexOf + substring). Comportamento identico all'implementazione originale.

### corridoioPrincipale.ts
- Rimossa la costante `activateCrystalCommand` (ora codice morto)
- Rimosso dalla lista `commands[]`
- Import `Command` rimosso dall'import (non più necessario)
