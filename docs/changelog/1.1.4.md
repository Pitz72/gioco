## Modifiche

### 1. Inventario formattato HTML — S3

**Problema:** La lista inventario era un'unica stringa piatta ("Stai trasportando: A, B, C.") senza struttura visiva.

**Soluzione:** `getInventarioHtml(state)` in `gameLogic.ts`:
- Header `INVENTARIO [N/∞]` in `var(--p-bright)` con letterSpacing
- Ogni item su riga separata con `► nome` (► in `var(--p-bright)`)
- Se inventario vuoto: messaggio in `var(--p-dim)`
- Output con `html: true` in `GameResponse` → `kind: 'html'` in `OutputLine`
- `types.ts`: aggiunto `html?: boolean` a `GameResponse`
- `App.tsx`: `outputKind = res.html ? 'html' : (res.typewriter ? 'typewriter' : 'text')`

---

### 2. Mappa ASCII progressiva — S4

**Problema:** Il comando `MAPPA` restituiva solo un placeholder "sistema in sviluppo".

**Soluzione:** `getMappa(state)` in `gameLogic.ts`:
- Layout ASCII a 14 stanze con connessioni `─` `│` `└`
- `[NOME]` per stanze visitate, `[???]` per inesplorate, `[*NOME*]` per posizione corrente
- Requires `state.visitedRooms` (nuovo campo) per determinare quali stanze mostrare

**Tracking stanze:** aggiunto in `processCommand` nel blocco `if (newState.location !== currentState.location)`:
```typescript
if (!newState.visitedRooms) newState.visitedRooms = [];
if (!newState.visitedRooms.includes(newState.location)) {
    newState.visitedRooms.push(newState.location);
}
```
- `types.ts`: `visitedRooms?: string[]` in `PlayerState` (opzionale per backward compat save file)
- `initialState` in `gameLogic.ts` e `INITIAL_PLAYER_STATE` in `App.tsx`: inizializzati con `['Plancia della Santa Maria']`
- `LEGGI` → `ANALIZZA` alias aggiunto in `normalizeCommand()` (utile anche per S6)

### 3. Schermata Crediti — S5

**Problema:** Nessuna schermata crediti accessibile dal menu.

- `borderRight: i < 7` aggiornato per 8 bottoni totali

### 4. Easter egg lettore ebook — S6

**Problema:** Il lettore di ebook nelle casse della Stiva era menzionato nella description di `casse_minerali` ma non era interattivo.

**Soluzione:** Item `lettore_ebook` aggiunto a `stiva.ts` (isFixed):
- Synonyms: `lettore`, `ebook`, `schermo rotto`, `libro`, `avventura testuale`, `castello di ghiaccio`
- `description`: visual description base (per ESAMINA)
- `onAnalyze` (flag `lettoreEsaminato`): rivela "CASTELLO DI GHIACCIO" di Paolo Vità, nota manoscritta dell'ex mercante, commento metafinzionale "stai giocando la stessa partita su scala galattica"
- Attivabile anche con `LEGGI LETTORE` grazie al nuovo alias `leggi → analizza`

---

### 5. Statistiche fine partita — S7

**Problema:** Il gioco terminava con il testo narrativo senza nessun feedback sulla completezza dell'esplorazione.

**Soluzione:** `getStats(state)` esportata da `gameLogic.ts`:
- Stanze esplorate (N/14), Echi (N/8), Traduzione (N%), Oggetti analizzati (N/7)
- Barre `█░` per ogni metrica
- Rating finale: ESPLORATORE CURIOSO / RECUPERATORE DI FRAMMENTI / ARCHIVISTA STELLARE / CUSTODE DELLA MEMORIA
- Iniettata in `submitCommand` di `App.tsx` prima del `gameOver` con `[PAUSE]` separator:
  ```typescript
  const finalResponse = rawResponse.gameOver
      ? { ...rawResponse, gameOver: getStats(newState) + '\n[PAUSE]\n' + rawResponse.gameOver }
      : rawResponse;
  ```
