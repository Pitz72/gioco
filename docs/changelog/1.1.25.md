# v1.1.25 — Bug fix + Error Boundary

## Bug 1: totalEchoes stale (8 → 11)
`gameLogic.ts` conteneva due costanti hardcoded rimaste a `8` dopo che
v1.1.19 aveva espanso gli echi da 8 a 11/14 stanze:
- `getStats()`: `const totalEchoes = 8` → `11`
- `getDiario()`: testo `state.echoes.length/8` → `/11`

Effetto visibile: un giocatore con tutti gli echi raccolta vedeva "11/8" nel
diario e una barra di progressione overflow nelle statistiche di fine partita.

## Bug 2: allItems ricalcolata ad ogni input
`processCommand()` conteneva `const allItems = Object.values(gameData).flatMap(r => r.items ?? [])`
dentro il corpo della funzione, traversando tutte le 14 stanze ad ogni pressione
di tasto. Spostato in una lazy cache a livello di modulo (`getAllItems()`),
costruita una sola volta al primo utilizzo.

## Error Boundary
Aggiunto `components/ErrorBoundary.tsx`: class component React che cattura
qualsiasi eccezione non gestita nel render tree. Se una stanza lancia un errore
imprevisto, l'app mostra una schermata di errore CRT con messaggio e pulsante
"RIAVVIA" invece di crashare silenziosamente.

`App.tsx`: `export default` ora esporta `AppWithBoundary` che wrappa `<App>`
in `<ErrorBoundary>`.

### Nota tecnica
React 19 non include `.d.ts` nel pacchetto npm. TypeScript risolve `Component`
come `any`, rendendo necessaria la dichiarazione esplicita
`declare readonly props: Readonly<Props>` nel corpo della classe per permettere
l'accesso a `this.props` senza errori TSC.
