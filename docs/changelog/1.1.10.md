## Modifiche

### 1. Fix regex `LEGGI` in Scriptorium (`scriptorium.ts`)

**Problema (P8):** Lo Scriptorium aveva un comando `^(leggi) (.+)$` che dava sempre la stessa risposta generica ("il testo alieno è incomprensibile a occhio nudo...") per qualsiasi oggetto. Questo creava l'impressione che `LEGGI LASTRA DATI` e `LEGGI PARETE` fossero equivalenti.

**Analisi:** Il problema era in gran parte già risolto dalla v1.1.x: `normalizeCommand` converte `leggi X` → `analizza X`, quindi:
- `LEGGI LASTRA DATI` → `analizza lastra dati` → item system → `onAnalyze()` → risposta specifica con percentuale traduzione
- `LEGGI PROIETTORI` → `analizza proiettori` → item system → `details` → risposta tecnica scanner
- `LEGGI PARETE` → `analizza parete` → fallback generico "non rilevi nulla di insolito"

Il regex `^(leggi) (.+)$` era pertanto **codice morto** (non veniva mai raggiunto), ma poteva essere fuorviante per chi legge il codice.

**Soluzione:**
- Rimosso il regex dead `^(leggi) (.+)$`
- Aggiunto handler per `^leggi$` (senza argomenti, l'unico caso non convertito dall'alias) che dà una risposta orientativa contestuale: "Le pareti sono coperte di globi olografici. Prova ANALIZZA PROIETTORI..."
