## Problema

I testi degli echi temporali erano duplicati in due posti:
1. `game/gameLogic.ts` — costante `ECHO_TEXTS` usata dal comando NOTA/DIARIO
2. `game/rooms/laboratoriRisonanza.ts` — testi hardcoded nel `onUse` del Sintonizzatore

La duplicazione aveva già causato disallineamento in v1.1.14: l'eco dell'Ingegnere Capo era stato espanso in `ECHO_TEXTS` con il dilemma morale, ma la copia in `laboratoriRisonanza.ts` mostrava ancora solo la vecchia riga di emergenza.

## Soluzione

### Nuovo file `game/echoData.ts`

Estratte entrambe le mappe in un file condiviso che non partecipa al ciclo di dipendenze:

```
echoData.ts   ← importato da gameLogic.ts e laboratoriRisonanza.ts
```

Il file esporta:
- **`ECHO_TEXTS`** — `Record<string, string>` con i testi di tutti gli echi (testi aggiornati e completi)
- **`LOCATION_TO_ECHO`** — `Record<string, string>` che mappa il nome della location all'echoId — include le 3 nuove location aggiunte in questa versione (Scriptorium, Stiva, Santuario del Silenzio)

La dipendenza circolare è evitata: `gameLogic.ts` importa da `gameData.ts`, `gameData.ts` importa da `laboratoriRisonanza.ts`; se `laboratoriRisonanza.ts` importasse da `gameLogic.ts` si chiuderebbe il ciclo. `echoData.ts` non importa nulla di locale, quindi non crea cicli.

### `game/gameLogic.ts`

- Rimossa la costante locale `ECHO_TEXTS` (17 righe)
- Aggiunto `import { ECHO_TEXTS } from './echoData'`

### `game/rooms/laboratoriRisonanza.ts`

- Aggiunto `import { ECHO_TEXTS, LOCATION_TO_ECHO } from '../echoData'`
- L'intera catena if/else di 35 righe nell'`onUse` del Sintonizzatore sostituita con 3 righe:
  ```typescript
  const echoId = LOCATION_TO_ECHO[state.location] ?? '';
  if (!echoId) return { ... };
  ```
