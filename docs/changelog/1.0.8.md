## Problema risolto: GR6 — `new RegExp()` invocato ad ogni comando

### Analisi

In `game/gameLogic.ts`, il loop sui comandi della stanza conteneva:

```typescript
for (const cmd of currentRoomData.commands) {
    const match = normalizedCommand.match(new RegExp(cmd.regex, 'i'));
    ...
}
```

Per ogni comando dell'utente, il motore JavaScript:
1. Itera su tutti i comandi della stanza corrente (mediamente 8–15 entries)
2. Per ciascuno, chiama `new RegExp(pattern, 'i')` — alloca un nuovo oggetto RegExp, compila il pattern, poi lo scarta

La compilazione di un'espressione regolare è l'operazione più costosa del ciclo. Essendo i pattern statici (non cambiano mai a runtime), questa compilazione ripetuta è uno spreco puro: lo stesso pattern viene ricompilato decine di migliaia di volte nel corso di una sessione di gioco.

### Correzione

Aggiunto a livello di modulo (eseguito una sola volta al caricamento) una `Map<string, RegExp>` e una funzione di accesso lazy:

```typescript
const regexCache = new Map<string, RegExp>();
const getCachedRegex = (pattern: string): RegExp => {
    let re = regexCache.get(pattern);
    if (!re) {
        re = new RegExp(pattern, 'i');
        regexCache.set(pattern, re);
    }
    return re;
};
```

Il punto di uso aggiornato:

```typescript
// Prima
const match = normalizedCommand.match(new RegExp(cmd.regex, 'i'));

// Dopo
const match = normalizedCommand.match(getCachedRegex(cmd.regex));
```

Ogni pattern viene compilato esattamente una volta — al primo utilizzo — e riutilizzato per tutta la vita del modulo. Le stanze hanno in totale ~100 pattern: dopo il warm-up, tutti i comandi successivi trovano le regex già pronte in cache.

- `npx tsc --noEmit` → zero errori ✅
- Comportamento del gioco invariato ✅
