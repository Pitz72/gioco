**Data:** 2026-03-20
**Tipo:** Fix criticità grave (GR2)
**Categoria:** Logica di gioco / Risoluzione item cross-room

## Problema risolto: GR2 — `USA X SU Y` non trovava oggetti nell'inventario

### Analisi

In `game/gameLogic.ts`, la gestione del comando `USA X SU Y` cercava la sorgente (`X`) **solo** tra gli item della stanza corrente:

```typescript
// Prima
const sourceItem = currentRoomData.items.find(i =>
    normalizeCommand(i.name) === sourceName ||
    i.synonyms.some(s => normalizeCommand(s) === sourceName)
);
```

Questo causava un comportamento errato ogni volta che `X` era un oggetto **raccolto** (presente nell'inventario). L'item, una volta preso, non è più nella lista `items` della stanza corrente, quindi la ricerca falliva silenziosamente, lasciando il giocatore con il messaggio generico "Non capisco quel comando" anche quando aveva l'oggetto giusto e l'obiettivo giusto.

**Esempio di failure:** il giocatore ha `Dispositivo Medico Alieno` nell'inventario ed è nella stanza giusta, ma `USA DISPOSITIVO SU CRISTALLO` non funzionava se il dispositivo non era tra gli item della stanza corrente.

### Correzione

La logica di ricerca è stata estesa in due fasi:

1. **Prima cerca nella stanza corrente** — per compatibilità con item fissi (non raccoglibili)
2. **Se non trovato, cerca nel registro globale** (`Object.values(gameData).flatMap(r => r.items ?? [])`) e verifica che l'item sia effettivamente nell'inventario del giocatore

```typescript
const matchesName = (i: { name: string; synonyms: string[] }, n: string) =>
    normalizeCommand(i.name) === n || i.synonyms.some(s => normalizeCommand(s) === n);

const allItems = Object.values(gameData).flatMap(r => r.items ?? []);

const sourceItem =
    currentRoomData.items.find(i => matchesName(i, sourceName)) ??
    allItems.find(i =>
        matchesName(i, sourceName) &&
        newState.inventory.some(inv => normalizeCommand(inv) === normalizeCommand(i.name))
    );
```

L'operatore `??` (nullish coalescing) garantisce che il fallback globale venga eseguito solo se la ricerca locale non trova nulla. Il cross-check `newState.inventory.some(...)` impedisce di trovare definizioni di item che il giocatore non ha ancora raccolto.

### Perché questo approccio

- **Nessun registro separato da mantenere**: usa `gameData` già disponibile come fonte di verità delle definizioni
- **Nessun accoppiamento aggiuntivo**: la logica è tutta in `gameLogic.ts`, le stanze non cambiano
- **Lazy**: la `flatMap` viene eseguita solo quando `USA X SU Y` viene invocato e il target non è nella stanza

### Verifica

- `npx tsc --noEmit` → zero errori ✅
- `USA DISPOSITIVO SU CRISTALLO` con `Dispositivo Medico Alieno` nell'inventario → attiva `onCombine` correttamente ✅
- Oggetti non in inventario → nessun match (cross-check inventario) ✅
