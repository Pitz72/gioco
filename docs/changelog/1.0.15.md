# Changelog v1.0.15 — `boot_screen.png` come Asset Vite con Fingerprint

**Data:** 2026-03-20

**Categoria:** Build / Cache invalidation

## Problema risolto: L2 — `boot_screen.png` non processata da Vite

### Analisi

In `BootScreen.tsx`, l'immagine della schermata di avvio era referenziata con un path hardcoded:

```tsx
<img src="./boot_screen.png" ... />
```

Il file si trovava in `public/`. Vite tratta `public/` in modo speciale: i file vengono copiati as-is nella directory di output senza alcuna elaborazione. Conseguenze:
- Nessun hash nel filename (`boot_screen.png` vs `boot_screen-Bx7kHq2A.png`)
- Nessuna cache invalidation automatica tra versioni: se l'immagine cambia, browser/Electron potrebbero servire la versione in cache
- Non beneficia della ottimizzazione asset di Vite (compressione, formato moderno)

### Correzione

**1. File spostato:** `public/boot_screen.png` → `assets/boot_screen.png`
Spostandolo fuori da `public/`, il file viene processato da Vite come qualsiasi altro asset importato.

**2. Import aggiunto in `BootScreen.tsx`:**

```typescript
import bootScreenImg from '../assets/boot_screen.png';
```

```tsx
// Prima
<img src="./boot_screen.png" ... />

// Dopo
<img src={bootScreenImg} ... />
```

Vite risolve l'import, copia il file in `dist/assets/` con hash nel filename (es. `boot_screen-Bx7kHq2A.png`) e sostituisce la variabile con l'URL corretto nel bundle.

**3. Dichiarazioni tipi aggiunte:**
Creato `vite-env.d.ts` con dichiarazioni `declare module '*.png'` (e jpg, svg, webp) per il type checker TypeScript — necessario perché TypeScript non conosce nativament i moduli immagine.

Aggiunto `"vite/client"` ai types in `tsconfig.json` per includere le definizioni built-in di Vite per asset statici.

### Verifica

- `npx tsc --noEmit` → zero errori ✅
- `public/boot_screen.png` rimosso (non più necessario lì) ✅
- `assets/boot_screen.png` presente ✅
