## Problema risolto: G1 — File barrel morto e incompleto

### Analisi del problema

Il file `game/rooms/index.ts` esportava un oggetto `allRooms` con la mappatura di tutte le stanze del gioco. Tuttavia presentava **due problemi gravi**:

**1. Non veniva importato da nessun file del progetto.**
La verifica con `grep -rn "from.*rooms/index"` su tutta la cartella `game/` non ha prodotto alcun risultato. `gameData.ts` importava ogni stanza direttamente dal proprio file sorgente:

```typescript
// gameData.ts — importa direttamente, MAI tramite index.ts
import { planciaRoom } from './rooms/plancia';
import { stivaRoom } from './rooms/stiva';
import { laboratoriRisonanzaRoom } from './rooms/laboratoriRisonanza';
// ... e così via per tutte le 14 stanze
```

**2. Mancava l'import di `laboratoriRisonanzaRoom`.**
L'oggetto `allRooms` esportato conteneva solo 13 stanze su 14. La stanza "Laboratori di Risonanza" era assente sia dagli import sia dall'oggetto esportato:

```typescript
// game/rooms/index.ts (ELIMINATO) — mancava questa riga:
// import { laboratoriRisonanzaRoom } from './laboratoriRisonanza';

export const allRooms = {
    // 13 stanze — "Laboratori di Risonanza" NON presente
    "Plancia della Santa Maria": planciaRoom,
    "Stiva": stivaRoom,
    // ...
    // "Laboratori di Risonanza": laboratoriRisonanzaRoom, ← ASSENTE
};
```

Se qualcuno avesse usato `allRooms` come fonte di verità per il mondo di gioco (ad esempio in un futuro refactoring o in un test), il gioco sarebbe stato privo di una stanza intera, causando crash durante l'Atto II (fase del Sintonizzatore e puzzle del cristallo).

### Soluzione applicata

**Eliminazione del file** `game/rooms/index.ts`.

La fonte di verità unica e corretta per il mondo di gioco è e rimane `game/gameData.ts`, che contiene tutte e 14 le stanze correttamente registrate. Mantenere un secondo registro parziale e non sincronizzato era una trappola per futuri sviluppatori.
