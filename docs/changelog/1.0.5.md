## Problema risolto: GR5 — Condizione `normalizedCommand === 'i'` mai vera

### Analisi

In `game/gameLogic.ts`, il check per il comando `inventario` era scritto così:

```typescript
if (normalizedCommand === 'inventario' || normalizedCommand === 'i') {
```

La condizione `normalizedCommand === 'i'` non può mai essere vera. La funzione `normalizeCommand()` (righe 41–42 dello stesso file) converte esplicitamente `'i'` in `'inventario'` **prima** che il controllo venga raggiunto:

```typescript
if (normalized === 'i' || normalized === 'inv') return 'inventario';
```

Quando si arriva alla riga 93, `normalizedCommand` è già stato trasformato: il valore `'i'` non esiste più. La seconda metà dell'`OR` è codice morto puro — non viene mai eseguita in alcun caso possibile.

### Correzione

```typescript
// Prima
if (normalizedCommand === 'inventario' || normalizedCommand === 'i') {

// Dopo
if (normalizedCommand === 'inventario') {
```

### Verifica

- `npx tsc --noEmit` → zero errori ✅
- Il comando `I` (abbreviazione) continua a funzionare correttamente perché viene convertito in `inventario` da `normalizeCommand()` ✅
- Nessuna modifica al comportamento del gioco ✅
