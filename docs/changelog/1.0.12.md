## Problema risolto: M4 — `echoes: string[]` popolato ma mai leggibile dal giocatore

### Analisi

Il `Sintonizzatore di Frequenza` registrava correttamente gli echi temporali nello stato del giocatore (`state.echoes.push(echoId)`), ma il campo `echoes` non era mai letto da nessun comando. Non esisteva alcun modo per il giocatore di riascoltare un eco già scoperto.

Conseguenze:
- Il giocatore poteva perdere informazioni narrative critiche (es. la menzione del "sistema trino" dal Capitano, chiave per capire il finale)
- Il game log di gioco scorreva verso l'alto: riascoltare un eco richiedeva di scrollare all'indietro nel terminale
- Il campo `echoes` di `PlayerState` era popolato inutilmente nei salvataggi JSON
- La feature era documentata nel MANUALE come elemento di lore, ma inaccessibile

### Correzione

**1. Registro globale degli echi** — aggiunto `ECHO_TEXTS: Record<string, string>` a livello di modulo in `gameLogic.ts`:

```typescript
const ECHO_TEXTS: Record<string, string> = {
    'echo_plancia': "CAPITANO: '...'",
    'echo_serra':   "XENO-BOTANICO: '...'",
    'echo_lab':     "INGEGNERE CAPO: '...'",
    'echo_arca':    "UFFICIALE MEDICO: '...'",
};
```

I testi sono speculari a quelli in `laboratoriRisonanza.ts` (callback `onUse` del sintonizzatore). La duplicazione è intenzionale: il registro serve solo alla consultazione, ed evita di accoppiare `gameLogic.ts` alla struttura interna delle stanze.

**2. Comando `ECHI`** — aggiunto in `processCommand()` prima del gestore `inventario`:

```typescript
if (normalizedCommand === 'echi' || normalizedCommand === 'mostra echi' || normalizedCommand === 'echi temporali') {
    if (newState.echoes.length === 0) {
        response = { description: "Non hai ancora rilevato nessun eco temporale...", eventType: null };
    } else {
        const echoList = newState.echoes
            .map((id, i) => `[ECO ${i + 1}] ${ECHO_TEXTS[id] ?? id}`)
            .join('\n\n');
        response = { description: `ECHI TEMPORALI REGISTRATI (${newState.echoes.length}):\n\n${echoList}`, eventType: 'echo' };
    }
    return { response, newState };
}
```

Il `eventType: 'echo'` attiva il suono "magico" appropriato per il lore.

**3. Aggiornata lista comandi** — `ECHI` aggiunto all'help text restituito da `AIUTO`.

### Verifica

- `npx tsc --noEmit` → zero errori ✅
- `ECHI` senza sintonizzatore usato → "Non hai ancora rilevato nessun eco temporale" ✅
- `ECHI` dopo aver usato il sintonizzatore → lista echi con ID e testo ✅
- `MOSTRA ECHI` e `ECHI TEMPORALI` → stessa risposta ✅
- Echi presenti nel salvataggio JSON vengono ripristinati correttamente al caricamento ✅
