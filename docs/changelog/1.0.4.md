## Problema risolto: G2 — Walkthrough Fase 1 completamente errato

### Analisi del problema

Il MANUALE.md (Sezione 4, Fase 1) descriveva questo percorso:

```
2.  **Stiva:**
    - USA TUTA (Indossala)
    - VAI EST (E) -> Plancia.
    - VAI SUD (S) -> Corridoio.    ← IMPOSSIBILE
```

**`VAI SUD` dalla Plancia non esiste.** Il file `game/rooms/plancia.ts` (riga 39) gestisce esplicitamente tutte le direzioni non valide:

```typescript
{ regex: "^((vai|va) )?(nord|n|sud|s|est|e)$",
  handler: () => ({ description: "Non puoi andare in quella direzione.", eventType: 'error' }) }
```

Un giocatore che seguiva il MANUALE veniva bloccato immediatamente dopo aver preso l'equipaggiamento nella Stiva, ricevendo un errore a ogni tentativo di raggiungere il Corridoio.

### Il percorso corretto (verificato contro il codice)

Il gioco richiede di attraversare **due stanze aggiuntive** (Scafo Esterno e Camera di Compensazione) con i relativi puzzle, che costituiscono l'intero **Atto I** del gioco:

| Stanza | Comando chiave | Logica nel codice |
|---|---|---|
| Stiva | `VAI SUD` | `scafoEsterno.ts`: exit sud in stiva se `isWearingSuit` |
| Scafo Esterno | `ANALIZZA SCAFO` | Setta flag `knowsAboutCrack` |
| Scafo Esterno | `USA TAGLIERINA SU CREPA` | Richiede `knowsAboutCrack` + taglierina in inventario |
| Scafo Esterno | `ENTRA` | Richiede flag `isHoleCut` |
| Camera di Compensazione | `ANALIZZA PANNELLO` | Scopre necessità di batteria |
| Camera di Compensazione | `USA BATTERIA SU PANNELLO` | Attiva porta |
| Camera di Compensazione | `APRI PORTA` | Accesso al Corridoio |

### Correzioni aggiuntive nella stessa sezione

- **Ordine degli oggetti in Stiva:** Il MANUALE diceva `PRENDI TUTA` prima di `PRENDI KIT`. La SOLUZIONE.md (verificata) dice `PRENDI KIT` prima e poi `USA KIT` (che dà la taglierina), poi `PRENDI TUTA`. L'ordine non è bloccante, ma è stato allineato alla guida ufficiale.
- **Chiave 2 (Seme):** `X PIANTE` → `ESAMINA PIANTE` per coerenza con il resto del documento (anche se `X` è un alias valido).
- **Fase 3, step 1:** Rimossa la dicitura confusa `S → E → S` (nel contesto errato). La Fase 3 ora descrive chiaramente dove si trova il giocatore al momento del ritorno al Corridoio.

## Problema risolto: M2 — Descrizione errata del Sintonizzatore di Frequenza

### Analisi del problema

La Sezione 2 (Comandi Avanzati) descriveva:

```
SINTONIZZATORE: Un comando speciale disponibile solo dopo aver trovato
l'oggetto specifico. Permette di ascoltare gli "Echi" del passato.
```

Questa descrizione implicava che `SINTONIZZATORE` fosse un **comando** che si digitava da solo. In realtà, il Sintonizzatore di Frequenza è un **oggetto** (`Item`) definito in `game/rooms/laboratoriRisonanza.ts` con una funzione `onUse`. Digitare `SINTONIZZATORE` da solo non produce risultati: il parser restituisce "Non capisco quel comando."

Il comando corretto è `USA SINTONIZZATORE`, esattamente come qualsiasi altro oggetto del gioco.

### Correzione applicata

**Sezione 2, Comandi Avanzati — prima:**
```
- SINTONIZZATORE: Un comando speciale disponibile solo dopo aver trovato
  l'oggetto specifico. Permette di ascoltare gli "Echi" del passato.
```

**Dopo:**
```
- USA SINTONIZZATORE: Usa il Sintonizzatore di Frequenza (oggetto che trovi
  nel gioco) per captare gli "Echi" del passato. Funziona come qualsiasi
  altro oggetto: devi prima trovarlo e raccoglierlo, poi usarlo con
  USA SINTONIZZATORE.
```

La Sezione 3 (Meccaniche — Gli Echi Temporali) è stata aggiornata allo stesso modo per coerenza.

## Problema risolto: M3 — `USA MONTACARICHI` non funziona

### Analisi del problema

La Sezione 4 (Fase 2, Chiave 3, step 3) diceva:

```
3.  Laboratori: Dal Corridoio, vai GIU (o USA MONTACARICHI).
```

`USA MONTACARICHI` viene processato dal sistema items come una chiamata a `item.onUse()` sull'oggetto `montacarichi`. Quell'oggetto non ha una funzione `onUse` definita (è `isFixed: true` senza handler), quindi il comando non produce il movimento verso il basso.

Il regex di movimento corretto in `corridoioPrincipale.ts` è:
```typescript
regex: "^((vai|va) )?(basso|giu|montacarichi)$"
```

`MONTACARICHI` da solo funziona (perché matcha il regex di movimento), `USA MONTACARICHI` no.

### Correzione applicata

**Prima:**
```
Dal Corridoio, vai GIU (o USA MONTACARICHI).
```

**Dopo:**
```
Dal Corridoio, vai GIU (o D, o VAI BASSO).
```

Aggiunta anche la nota sulle abbreviazioni `U`/`D` per Alto/Basso nella tabella dei comandi base (mancavano completamente).
