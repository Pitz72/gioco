# v1.2.0 — Stabilizzazione sistema comandi

**Data:** 2026-03-21
**Tipo:** Qualità e stabilità — risposta del parser più coerente e completa

---

## Contesto

Dopo la mappatura totale dei comandi (v1.1.40), sono stati identificati cinque punti di fragilità nel sistema di parsing e risposta. Nessuno di essi causa blocchi al completamento del gioco, ma ciascuno produce comportamenti incoerenti o risposte grammaticalmente scorrette in scenari legittimi. Tutti e cinque sono stati risolti con interventi chirurgici sui tre file interessati.

---

## S1 — TOCCA sul Ponte di Comando: handler mancanti

### Problema
Il Ponte di Comando era l'unica delle 15 stanze priva di handler `TOCCA`. Qualunque comando `TOCCA [oggetto]` o `TOCCA` senza argomento cadeva nel fallback globale con un messaggio grammaticalmente scorretto e narrativamente vuoto.

### Fix
Aggiunti quattro handler in `ponteDiComando.ts`:
- `^(tocca) (struttura|stella|proiettore|centro)$` — risposta contestuale (ologramma attivo / spento)
- `^(tocca) (postazione|console|pannello|controlli|depressione)$` — risposta contestuale (console attiva / inerte)
- `^(tocca) (porta|porta circolare|meccanismo)$` — risposta contestuale (porta aperta / chiusa)
- `^tocca$` — hint generico con lista oggetti toccabili

### File modificato
- `game/rooms/ponteDiComando.ts`

## S2 — ANALIZZA sulla Plancia: risposte non tematiche

### Problema
`pannello_comandi` e `radio_comunicazioni` non avevano `details`, quindi `ANALIZZA PANNELLO` e `ANALIZZA RADIO` restituivano il fallback generico dell'item system («Analizzi [nome], ma non rilevi nulla di insolito»). La risposta era funzionalmente corretta ma narrativamente vuota — la Plancia è una stanza umana e merita risposte che rinforzino questo contrasto con il relitto alieno.

### Fix
Aggiunti due campi `details` tematici:
- `pannello_comandi.details` — descrive le letture nominali dei sistemi umani + anomalia allarme prossimità
- `radio_comunicazioni.details` — descrive il silenzio totale su tutte le frequenze, nessuna emissione aliena

### File modificato
- `game/rooms/plancia.ts`

## S3 — TOCCA X globale: fallback assente

### Problema
`TOCCA [oggetto]` non era incluso nel Percorso 2 dell'item system (`^(esamina|guarda|analizza|prendi|raccogli|usa|attiva) (.+)$`). Se una stanza non aveva un handler specifico per quell'oggetto, il comando cadeva nel blocco `genericUsaSingle` che:
1. Includeva `tocca` nella regex ma produceva un messaggio («Cosa vuoi tocca con 'X'?») grammaticalmente scorretto
2. Suggeriva `USA X SU ...` che non ha senso per TOCCA

### Fix
Aggiunto in `gameLogic.ts` un handler dedicato `^tocca (.+)$` **prima** del blocco `genericUsaSingle`. Risposta: «Sfioreresti [oggetto], ma non c'è nulla di particolare da notare al tatto.» — neutra, coerente, non fuorviante.

### File modificato
- `game/gameLogic.ts`

## S4 — Grammatica nei fallback verbali

### Problema
Il blocco `genericUsaSingle` usava `${action}` (forma imperativa: «usa», «apri», «leggi») nella stringa di risposta, producendo frasi come «Cosa vuoi apri con 'X'?» e «Cosa vuoi leggi con 'X'?» grammaticalmente scorrette.

Contestualmente, il blocco includeva `tocca` che è stato estratto (S3).

### Fix
Sostituiti `action` con le forme infinitive tramite lookup `verbInfinitive`:
```
usa → usare · attiva → attivare · apri → aprire · leggi → leggere
```
La voce `tocca` è stata rimossa dalla regex (ora gestita da S3).

### File modificato
- `game/gameLogic.ts`

## S5 — Lookup inventario: match più preciso nei fallback generici

### Problema
I blocchi fallback `genericUsaSuY` e `genericUsaSingle` usavano `normalizeCommand(inv).includes(srcName)` — un confronto substring che:
- Permetteva a input parziali ambigui (es. «la») di trovare corrispondenze indesiderate in inventario
- Era incoerente con il resto del sistema (Percorso 1 e 2 usano `=== normalizeCommand(i.name)`)

### Fix
Il confronto è stato sostituito con una funzione locale che accetta solo:
1. Corrispondenza esatta normalizzata: `n === srcName`
2. Corrispondenza prefisso: `n.startsWith(srcName + ' ')`

Questo consente ancora «cristallo» di trovare «Cristallo Dati Attivato» (prefisso valido) ma esclude match ambigui da singole sillabe o stringhe brevi.

### File modificato
- `game/gameLogic.ts`
